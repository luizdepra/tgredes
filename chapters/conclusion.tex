\chapter{Conclusão}

	Neste trabalho comparamos o desempenho de busca e inserção de registros com
	múltiplas chaves em árvores multidimensionais, a árvore \textit{K-D} e a
	árvore \textit{K-D-B}, e um banco de dados relacional, o MySQL. Os
	resultados obtidos através dos  experimentos em memória primária apontaram
	que a árvore \textit{K-D} teve, como esperado, resultados melhores que o
	MySQL. Na comparação das estruturas em memória secundária, os testes
	iniciais mostraram que a árvore \textit{K-D-B} foi mais eficiente que o
	MySQL. Porém, com alguns ajustes no banco de dados, a diferença entre os
	resultados foi reduzida. 

	Trabalhos futuros podem incluir o desenvolvimento de uma interface web para
	a ferramenta, que permita a configuracão, monitoração e consulta
	multidimensional.  Além disso, pode ser acoplado um módulo para geracão de
	gráficos dos recursos monitorados. Investigando as estruturas usadas pelo
	MySQL para organizar os índices, foi encontrada como referência a árvore
	\textit{R} \cite{guttman}, que é utilizada para armazenar índices espaciais.
	A árvore \textit{R} é uma estrutura de árvore balanceada semelhante à árvore
	\textit{B}, em que cada nó da árvore é associado a uma faixa de fronteira
	semelhante a árvore \textit{K-D}. Essa estrutura é bastante utilizada para
	armazenar índices de estruturas multidimensionais. Novos experimentos
	poderiam ser realizados com a árvore \textit{R}. Por fim, a integracão da
	ferramenta a um sistema de grade comercial com acordos de nível de serviço
	também pode ser tema para trabalhos futuros.
