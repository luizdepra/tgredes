\chapter{Conclusão}

  Neste trabalho comparamos o desempenho de busca e inserção de registros com múltiplas chaves em árvores multidimensionais, \textit{K-D} e \textit{K-D-B}, e um banco de dados relacional, o MySQL. Os resultados obtidos através dos  experimentos em memória primária apontaram que a árvore \textit{K-D} teve resultados melhores que o MySQL. Na comparação das estruturas em memória secundária, os testes iniciais mostraram que a árvore \textit{K-D-B} foi mais eficiente que o MySQL. Mas, com alguns ajustes no banco de dados, a diferença entre os resultados foi reduzida. Os resultados nos mostraram que, as estruturas de dados podem ser mais eficientes que sistemas de armazenamento complexos.

  Investigando as estruturas usadas pelo Mysql para organizar os índices, foi encontrado referência a árvore-\textit{R}, que é utilizada para armazenar índices espaciais. A árvore-\textit{R} é uma estrutura de árvore balanceada parecida com a \textit{B}, em que cada nó da árvore é associada uma caixa de fronteira semelhante a árvore \textit{KD}. Essa estrutura é bastante utilizada para armazenar índices de estruturas multidimensionais. 
  
  Os experimentos realizados foram simples. Novos experimentos mais detalhados poderiam mostrar de uma forma mais clara os resultados, como por exemplo poderia ser utilizada as ferramentas para armazenamento de dados multidimensionais disponibilizada pelo SGBD. Além disso, aprofundar o estudo das ferramentas envolvidas melhoraria o entendimento do seu funcionamento e a interpretação dos resultados.

  Trabalhos futuros incluem o desenvolvimento de uma interface Web para a ferramenta, permitindo a configuracão, monitoração e consulta multidimensional. Também está previsto o acoplamento de um módulo para geracão de gráficos dos recursos monitorados. Por fim, a integracão da ferramenta a um sistema de grade comercial com acordos de nível de serviço é também objetivo do trabalho.
