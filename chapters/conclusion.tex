\chapter{Conclusão}

  Neste trabalho comparamos o desempenho de busca e inserção de registros com múltiplas chaves em árvores multidimensionais, a árvore \textit{K-D} e a árvore \textit{K-D-B}, e um banco de dados relacional, o MySQL. Os resultados obtidos através dos  experimentos em memória primária apontaram que a árvore \textit{K-D} teve resultados melhores que o MySQL. Na comparação das estruturas em memória secundária, os testes iniciais mostraram que a árvore \textit{K-D-B} foi mais eficiente que o MySQL. Porém, com alguns ajustes no banco de dados, a diferença entre os resultados foi reduzida. Os resultados nos mostraram que, as estruturas de dados podem ser mais eficientes que sistemas de armazenamento complexos.

  Investigando as estruturas usadas pelo MySQL para organizar os índices, foi encontrado referência a árvore \textit{R}, que é utilizada para armazenar índices espaciais. A árvore \textit{R} é uma estrutura de árvore balanceada semelhante a árvore \textit{B}, em que cada nó da árvore é associado a uma caixa de fronteira semelhante a árvore \textit{K-D}. Essa estrutura é bastante utilizada para armazenar índices de estruturas multidimensionais. 
  
  Os experimentos realizados foram simples. Novos experimentos mais detalhados poderiam mostrar de uma forma mais clara os resultados. Por exemplo, poderiam ser utilizadas as ferramentas para armazenamento de dados multidimensionais disponibilizada pelo SGBD usando a árvore \textit{R}. Além disso, aprofundar o estudo das ferramentas envolvidas melhoraria o entendimento do seu funcionamento e a interpretação dos resultados.

  Trabalhos futuros podem incluir o desenvolvimento de uma interface web para a ferramenta, permitindo assim a configuracão, monitoração e consulta multidimensional. Além disso, poderia ser acoplado um módulo para geracão de gráficos dos recursos monitorados. Por fim, a integracão da ferramenta a um sistema de grade comercial com acordos de nível de serviço podem ser temas para trabalhos futuros.
