\chapter{Introdução}

  As redes de computadores e aplicações distribuídas têm se tornado ferramentas indispensáveis para as organizações e pessoas. Com o aumento exponencial do tráfego de informações devido ao surgimento de novas aplicações, é fundamental manter a qualidade dos serviços de rede. Além disso, novas tecnologias e padrões têm aumentado a heterogeneidade dos dispositivos conectados às redes. Por esses motivos, controlar e monitorar um ambiente de rede é uma tarefa cada vez mais complexa e necessária.
	
  A gerência de redes \cite{NetworkManagement} consiste no controle e monitoramento dos componentes de uma rede, que podem ser físicos ou lógicos, para atingir uma melhor utilização dos  recursos de rede e assegurar um certo nível de serviço. Os níveis de serviço são definidos como contratos de níveis de serviços (SLA - \textit{Service Level Agreements}). Esses contratos são assinados entre provedores e clientes de serviços. Os SLAs explicitam formalmente o que um provedor de serviço irá entregar aos seus clientes. Além disso, os SLAs definem os critérios de aceitabilidade de um serviço, as formas de verificar se os critérios de aceitabilidade estão sendo assegurados, além de incluir os compromissos tanto do provedor como do cliente.

  A arquitetura geral dos sistemas de gerência de rede é dividida em quatro partes básicas: elementos gerenciados, estações de gerência, protocolos de gerência e informações de gerência \cite{GerenciaRede}. Os elementos gerenciados são os componentes da rede que precisam operar adequadamente para que toda a rede ofereça os serviços para quais foi projetada. Cada um dos elementos gerenciados possui um software especial chamado agente, o qual permite que os elementos gerenciados sejam controlados e a monitorados remotamente. As estações de gerência são responsáveis pelo controle e monitoramento dos elementos gerenciados. Também munidas de um software especial, chamado gerente. As informações trocadas entre os agentes e gerentes são chamadas de informações de gerência, e a transmissões desses dados é feita através dos protocolos de gerência. Um protocolo de gerência bastante utilizado em redes TCP/IP é o SNMP (\textit{Simple Network Management Protocol}).
  
  O objetivo deste trabalho é comparar o desempenho do uso de árvores multidimensionais com o uso de bancos de dados relacionais, em especial o MySQL, no armazenamento de dados obtidos a partir da monitoração de um rede. Para isso, foram realizados experimentos de busca e inserção em árvore \textit{K-D}, árvore \textit{K-D-B} e banco de dados MySQL.

  Os resultados obtidos a partir dos experimentos de comparação entre a árvore K-D e o MySQL mostraram a árvore foi mais eficiente tanto na inserção quanto na busca. O mesmo ocorreu inicialmente com a comparação entre a árvore K-D-B e o MySQL. Mas com algumas alterações nas configurações do MySQL, em alguns experimentos o banco de dados teve os melhores resultados.

  Este trabalho está organizado da seguinte forma. O capítulo 2 apresenta a pesquisa multidimensional incluindo definições básicas e as estruturas de dados utilizadas no trabalho. O capítulo 3 descreve os experimentos realizados, e resultados obtidos. O trabalho finaliza no capítulo 4 com a conclusão.
